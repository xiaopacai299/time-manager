# 移动端打包说明

当前用 **Expo Go** 做日常开发，本地不保留 `android/` 工程。要生成 **可安装的 APK / AAB（或 iOS IPA）**，推荐 **EAS Build（云端打包）**；若必须在本机编译，可用 **`expo prebuild` + Android Studio**（需自行安装 JDK、Android SDK、NDK 等）。

---

## 一、推荐：EAS Build（云端）

适合：上架商店、发安装包、真机安装独立 App（不依赖 Expo Go）。

### 1. 准备

- 注册 [expo.dev](https://expo.dev) 账号。
- 在 **`packages/mobile`** 目录下执行后续命令（本项目的 `app.json` 在此目录）。

### 2. 安装 CLI 并登录、初始化

```powershell
cd D:\work\time-manager\packages\mobile
pnpm add -D eas-cli
pnpm exec eas login
pnpm exec eas build:configure
```

`build:configure` 会生成 **`eas.json`**，并在 `app.json` 中写入 **`extra.eas.projectId`**，请勿随意删除。

### 3. 发起构建

**Android（默认多为 AAB，适合上架 Google Play）：**

```powershell
pnpm exec eas build -p android --profile production
```

**若需要 APK（内测、直接安装）：**

在 `eas.json` 中增加 `preview` 等 profile，将 Android 构建类型设为 `apk`。具体字段以 [Expo 文档：Android builds](https://docs.expo.dev/build-reference/android-builds/) 为准。

**iOS：**

```powershell
pnpm exec eas build -p ios --profile production
```

（需 Apple 开发者账号与证书配置，按 EAS 向导完成。）

### 4. Monorepo（pnpm）注意

仓库为 pnpm workspace，移动端依赖 **`@time-manger/shared`**。一般在 **`packages/mobile`** 下执行 `eas build` 即可。

若云端构建报错找不到 workspace 包，可在 `eas.json` 对应 profile 的 `env` 中增加（以实际报错与 [Expo Monorepo 指南](https://docs.expo.dev/guides/monorepos/) 为准）：

```json
"env": {
  "EXPO_USE_METRO_WORKSPACE_ROOT": "1"
}
```

### 5. 取包

构建在 Expo 云端完成后，终端或 [expo.dev](https://expo.dev) 控制台会提供 **下载链接**（APK / AAB / IPA）。

---

## 二、备选：本地打包

若不想使用云端构建，可在 **`packages/mobile`** 执行：

```powershell
npx expo prebuild
```

会重新生成 **`android/`**（及可选 **`ios/`**），再用 **Android Studio** 打开 `android` 工程完成签名与出包。此方式需要本机安装 **JDK、Android SDK、NDK** 等，与「仅用 Expo Go 开发」的路径不同。

---

## 三、与 Expo Go 的关系

| 方式 | 用途 |
|------|------|
| **Expo Go + `pnpm start`** | 日常开发、热更新 |
| **EAS Build 产物（APK/AAB/IPA）** | 真机安装、内测、商店上架 |
| **`expo prebuild` + 本地 Gradle / Xcode** | 离线构建或需改原生代码时 |

---

## 四、一句话流程

日常开发继续用 Expo Go；需要安装包时：在 **`packages/mobile`** 下执行 **`eas login` → `eas build:configure` → `eas build -p android`**（或 iOS），按终端与网页提示下载构建结果即可。
