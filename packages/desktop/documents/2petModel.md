# Pet Model V1 设计说明

## 目标

- 将时间管理应用改造成桌面宠物形态，常驻桌面、可交互、低打扰。
- 利用已有时间采集结果（当前应用、连续使用、休息累计）进行主动提醒。
- 保持后续可扩展：动画资源、任务系统、语音/聊天能力。

## 架构总览

- 主进程：`electron-main.js`
  - 创建透明无边框宠物窗口（置顶、可隐藏、可穿透）
  - 维护托盘菜单（显示/隐藏、穿透开关、退出）
  - 保存宠物窗口位置与开关状态到本地
  - 承担 IPC：窗口拖拽、状态读取、开关切换
- 时间采集：`main/time-monitor-service.js` + `main/activity-engine.js`
  - 持续采样前台应用与 idle 信息
  - 汇总 per-app 时长、连续使用时长、休息累计
  - 推送实时快照给渲染层
- 渲染层：`src/App.jsx` + `src/App.css`
  - 宠物 UI（头像、气泡、交互按钮）
  - 拖拽行为（pointer 事件 + IPC）
  - 主动交互文本（依据时间状态变化）
  - 可切换小面板查看核心统计

## 交互模型

### 用户交互

- 拖拽宠物：按住宠物头像移动，窗口随鼠标位置改变。
- 穿透开关：切换窗口是否接收鼠标。
- 面板开关：隐藏/显示统计信息区域。

### 宠物主动交互

- 连续使用超过 50 分钟：提醒短休息。
- 休息累计达到阈值（如 5 分钟）：正向反馈。
- 处于休息中：宠物进入“安静陪伴”状态。
- 常态：显示当前专注应用。

## 状态字段（V1）

- `petState.clickThrough`: 是否鼠标穿透。
- `petState.showStatsPanel`: 是否展示统计面板。
- `snapshot.current.*`: 当前应用信息（名称、窗口标题、进入时刻、空闲状态）。
- `snapshot.continuousUseMs`: 连续使用时长。
- `snapshot.breakCompletedMs`: 休息累计时长。
- `snapshot.perAppToday[]`: 今日应用时长排名。

## 数据流

1. 主进程定时采样 -> 生成 `sample`
2. `activity-engine` 汇总 -> 生成 `snapshot`
3. 主进程 `webContents.send` 推送 `snapshot`
4. 渲染层依据 `snapshot` 更新宠物表情、文案、统计
5. 用户交互通过 IPC 反向控制主进程窗口

## 扩展建议（V2+）

- 宠物动画资源层：Lottie/Sprite 切换不同 mood。
- 规则中心：把“提醒阈值”抽为可配置策略。
- 多宠物皮肤：角色/主题包加载。
- 任务系统：把待办与专注番茄钟接到宠物行为。
- 交互记忆：记录用户偏好（提醒频率、静音时段）。

## V2 已实现项

- 托盘图标资源：`assets/tray-icon.svg`，主进程读取并加载到系统托盘。
- 右键宠物菜单：支持模式切换、穿透切换、统计面板显示切换与退出。
- 双击切换模式：双击宠物头像可在紧凑/展开模式间切换。
- 动作系统：
  - `idle`（待机）
  - `nod`（点击触发点头）
  - `sleep`（休息时困倦）
  - `celebrate`（双击触发庆祝）
  - `warn`（连续工作提醒）
