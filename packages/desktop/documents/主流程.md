Electron：负责窗口、托盘、IPC、定时器、通知等「壳」

IPC = Inter-Process Communication（进程间通信）。
在 Electron 里，主进程和渲染进程是分开的，不能直接互相调用函数，所以要靠 IPC 传消息。

你项目里有两种常见形式：

请求-响应（像调用接口）

渲染进程：ipcRenderer.invoke('xxx')
主进程：ipcMain.handle('xxx', handler)
例子：pet:get-state、worklist:add、favorites:get-list
广播推送（主进程主动通知）

主进程：webContents.send('time-stats:update', payload)
渲染进程：ipcRenderer.on('time-stats:update', callback)
例子：时间采样更新就是这种
通俗讲：

invoke/handle = “我问你一个问题，你回我一个答案”
send/on = “我有新消息，主动推给你”

2) 渲染进程是哪个？
渲染进程就是每个 BrowserWindow 里跑页面的那个进程（Chromium 页面进程）。

你项目里主要有这些渲染进程：

宠物主窗口（createMainWindow 创建）
统计窗口（openStatsDetailWindow 创建，hash 是 #stats）
收藏夹窗口（favorites module 创建，hash 是 #favorites）
工作清单窗口（worklist module 创建，hash 是 #worklist）
这些窗口加载的都是前端页面（Vite 页面 / 打包后的 dist/index.html），所以都属于渲染进程。

BrowserWindow：electron提供的创建渲染进行，页面窗口的方法
