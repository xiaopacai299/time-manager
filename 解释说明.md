## 整体架构

[Electron 桌面端]
   │ 1. 用户操作（比如同步、保存日记、拉取工作清单）
   ▼
[API Client]
   │ 2. 发 HTTP 请求（例如 /sync/push, /sync/pull）
   ▼
[Node.js 运行的 Server 项目]
   │ 3. 路由层接收请求（如 routes/sync.ts）
   │ 4. 做鉴权/参数校验/业务规则
   │ 5. 调用 Prisma Client
   ▼
[Prisma]
   │ 6. 把 JS/TS 的查询调用转成 SQL
   ▼
[PostgreSQL（通常跑在 Docker 容器里）]
   │ 7. 执行 SQL，返回结果
   ▲
   └──────── 8. 原路返回给前端，界面更新

Prisma干的两件大事
Schema 建模：在 schema.prisma 定义数据模型（表结构）
Client 调用：在代码里写 prisma.user.findMany() 这类 JS/TS API，而不是手写大量 SQL

### --filter 参数解释
--filter 是 pnpm 包管理器的一个重要参数，用于在 monorepo 项目 中指定要操作的具体包。

- 作用 ：限制命令只作用于特定的包，而不是整个项目
- 使用场景 ：当项目包含多个子包时，可以只对需要的包执行命令，提高效率
- 语法 ： pnpm --filter <package-name> <command>

## 数据库
1.查看数据库
pnpm dlx prisma@5.22.0 studio --schema packages/server/prisma/schema.prisma
2.pgSQL不是跑在本机上，而是跑在docker上。docker desktop就是容器运行平台。
数据库本身是容器，所以pgSQL是容器。
3.docker
启动和管理容器（比如 postgres 容器）
提供网络映射（把容器里的 5432 端口映射到你本机）
提供数据卷（数据库文件持久化）
让 docker compose up 这类命令真正运行起来
4.项目中用 Docker 按 compose 文件帮你装并启动了一个 Postgres 容器，image: postgres:16指定镜像，docker会从网上拉这个镜像，起一个独立容器，跑真正的Postgres服务。

5.Docker Desktop 与 compose 对照表（新手速查）

| 你在项目里写的 compose | Docker Desktop 里看到的东西 | 含义 |
|---|---|---|
| `services.postgres` | Containers 里的 `...postgres...` 容器（Running/Stopped） | 真正跑 PostgreSQL 的容器 |
| `image: postgres:16` | Images 里的 `postgres:16` | Postgres 软件镜像，容器由它创建 |
| `ports: "5432:5432"` | 容器详情里的 Port 映射 | 容器 5432 暴露到本机 `localhost:5432` |
| `volumes: postgres_data:/var/lib/postgresql/data` | Volumes 里的 `postgres_data` | 数据持久化位置，重建容器后数据通常还在 |
| `environment: POSTGRES_USER/POSTGRES_PASSWORD/POSTGRES_DB` | 容器启动时注入环境变量（桌面端可在容器详情看到） | 初始化数据库用户、密码和默认库 |
| `services.server` | Containers 里的 `...server...` 容器（如果你起了它） | 后端也可以容器化运行；本地 `pnpm run server:dev` 则不依赖这个容器 |
| `DATABASE_URL=...@postgres:5432/...`（server 容器内） | 同一 Compose 网络内服务名互通 | 容器内用 `postgres` 主机名；本机启动 server 时要用 `localhost` |

常用操作：
- 只启动数据库：`pnpm run db:up`（等价于 `docker compose -f packages/server/docker-compose.yml up -d postgres`）
- 看容器是否正常：打开 Docker Desktop -> Containers，看 postgres 是否 Running
- 看表和数据：推荐 `pnpm --filter @time-manger/server exec prisma studio` 或使用 DBeaver/pgAdmin 连 `localhost:5432`
